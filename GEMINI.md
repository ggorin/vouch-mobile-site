# Vouch Mobile Site - Project Context

## Project Overview
This project is the marketing website for **Vouch**, a mobile carrier service. It is a static site built using **Eleventy (11ty)**, utilizing **Nunjucks** templates for structure and **Markdown** for content (blog posts).

## Tech Stack & Key Technologies
*   **Static Site Generator:** Eleventy (v3.x)
*   **Templating Engine:** Nunjucks (`.njk`)
*   **Content Format:** Markdown (`.md`) & Nunjucks
*   **Styling:** CSS (located in `src/css/`)
*   **Scripting:** Vanilla JavaScript (located in `src/js/`)
*   **Package Manager:** NPM

## Build & Run Commands
The following scripts are defined in `package.json`:

*   **Development Server:**
    ```bash
    npm start
    ```
    Runs `eleventy --serve`. Watches for changes and live-reloads.

*   **Production Build:**
    ```bash
    npm run build
    ```
    Generates the static site into the `_site/` directory.

*   **Clean Build Artifacts:**
    ```bash
    npm run clean
    ```
    Removes the `_site` directory.

## Directory Structure
*   **`src/`**: The source directory for all content and code.
    *   **`_data/`**: Global data files (JSON). Contains `competitors.json`, `faqs.json`, `site.json`, `states.json`.
    *   **`_includes/`**: Nunjucks layouts and partials (e.g., `base.njk`, `header.njk`, `footer.njk`).
    *   **`blog/posts/`**: Markdown files for blog posts.
    *   **`css/`, `js/`, `images/`**: Static assets that are passed through to the build.
    *   **`compare/`, `coverage/`**: Section specific templates.
*   **`_site/`**: The compiled output directory (generated by the build).
*   **`.eleventy.js`**: Main configuration file for Eleventy. Handles plugins, passthroughs, collections, and filters.

## Development Conventions

### Collections
The site uses Eleventy collections to organize content:
*   **`posts`**: Generated from `src/blog/posts/*.md`. Sorted by date.
*   **`states`**: Generated from items where `layout` is `state.njk`.
*   **`competitors`**: Generated from items where `layout` is `compare.njk`.

### Adding Content
1.  **Blog Posts**: Add a new `.md` file in `src/blog/posts/` with the appropriate front matter.
2.  **Pages**: Create `.njk` files in `src/` or subdirectories. Use layouts from `_includes`.
3.  **Static Assets**: Place images, CSS, or JS in their respective folders in `src/`. They are automatically copied to the output.

### Configuration (`.eleventy.js`)
*   **Passthrough Copy**: Configured for `css`, `js`, `images`, `favicon.png`, and `robots.txt`.
*   **Filters**:
    *   `readableDate`: Formats dates for display (e.g., "January 1, 2024").
    *   `isoDate`: Formats dates for machine reading/sitemaps.
    *   `slug`: Generates URL-friendly slugs.
*   **Shortcodes**:
    *   `year`: Returns the current year (useful for footers).
